Key Terms — Defined Once
Embedding

A fixed-length numerical vector that captures the semantic meaning of text. This app uses all-MiniLM-L6-v2 (384 dimensions). Semantically similar texts produce vectors close together in vector space.

Cosine Similarity

Measures the angle between two vectors: cos θ = (A·B) / (|A||B|). Returns 1.0 for identical direction (most similar), 0 for orthogonal (unrelated). The primary ranking signal in Naive, Hybrid, and Advanced RAG.

Dense Retrieval (Vector Search)

Embed both query and documents into vector space, then rank by cosine similarity. Captures semantic meaning — "automobile" matches "car" — but can miss exact keyword terms not captured by the embedding model.

Sparse Retrieval — BM25 / TF-IDF

Ranks documents by term frequency (TF) weighted by inverse document frequency (IDF). Exact keyword match — if the query contains "RFC 2119" the BM25 path finds it even if no vector-similar chunk exists. Used in Hybrid, Advanced, and Agentic RAG.

Reciprocal Rank Fusion (RRF)

Merges multiple ranked lists without needing score normalisation. Formula: score = Σ 1/(rank + k), k=60 (standard constant). Documents appearing in multiple lists are boosted; k=60 prevents top-1 from dominating.

Chunking Strategies

Fixed-size — split every N characters regardless of content (Naive / Hybrid). Sentence-aware — split at sentence boundaries to preserve meaning. Recursive — try paragraph → sentence → word boundaries in order. Semantic — group sentences with similar embeddings into one chunk.

HyDE — Hypothetical Document Embeddings

Ask the LLM to write a hypothetical answer, then embed that answer instead of the original question. The hypothetical answer lives in "document space" — closer to real documents — which improves retrieval recall for complex queries.

Cross-encoder Reranking

A bi-encoder (independent embeddings) retrieves candidates fast. A cross-encoder then scores each query–chunk pair jointly, seeing both at once for higher precision. Slower — O(candidates) LLM calls — but surfaces the best chunks. Used in Advanced RAG.

Knowledge Graph + BFS

Documents modelled as a graph: each chunk is a node, sequential chunks are linked by edges. Retrieval starts with vector-matched seed nodes, then expands via BFS (Breadth-First Search) to pull in neighbouring chunks — capturing multi-hop context missed by flat retrieval.

Agentic Loop — Think → Act → Observe

The LLM thinks about what information it needs, acts by calling a search tool, then observes the results and decides whether to search again or answer. Capped by max_search_rounds to bound latency and cost.

Context Compression

Post-retrieval clean-up before sending chunks to the LLM: deduplication removes near-identical chunks, summarisation condenses long chunks, reordering places the most relevant chunk first (lost-in-the-middle mitigation). Applied in Advanced RAG.

LLM-as-Judge Evaluation

Use an LLM to score RAG output on four dimensions: Faithfulness (no hallucinations), Answer Relevancy (on-topic), Context Utilization (right chunks retrieved), Correctness (vs ground truth, optional). More flexible than BLEU/ROUGE for open-ended answers.

Decision Guide — Which Strategy to Use
Strategy Use When Avoid When Relative Cost
1. Naive RAG Prototype / proof-of-concept; simple, well-structured docs; latency-critical Complex multi-part questions; exact keyword queries; production systems Lowest
2. Hybrid RAG Mixed semantic + keyword queries; technical docs with exact terms (IDs, codes) Pure narrative docs with no exact-term queries; memory-constrained environments Low
3. Advanced RAG Production; high-stakes answers; ambiguous queries; when precision matters Simple queries where rewriting adds no value; tight latency or budget constraints Medium
4. Graph RAG Sequential / relational docs (manuals, legal, research papers); multi-chunk answers Unordered, self-contained chunks; no meaningful adjacency between chunks Medium
5. Agentic RAG Multi-hop questions; open-ended research; when the agent needs to self-correct Latency-sensitive apps; simple factual lookup; cost-constrained environments Highest

Key Takeaway

Each strategy is a strict superset of the previous one — Hybrid adds keyword retrieval to Naive, Advanced adds query processing and reranking to Hybrid, Graph adds structural traversal to Advanced, and Agentic adds an autonomous multi-step loop over all of them. Start with Naive to prove the pipeline works, move to Hybrid when exact-term queries fail, add Advanced when precision matters, use Graph when answers span multiple sequential chunks, and reach for Agentic only when the question complexity justifies the latency cost.
