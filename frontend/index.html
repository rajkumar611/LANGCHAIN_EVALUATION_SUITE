LangChain & LangGraph Fundamentals

Production-grade patterns for building AI applications with LLM orchestration, multi-agent systems, and stateful workflows.

What is LangChain?

LangChain is a framework for developing applications powered by large language models (LLMs). It provides abstractions for composable building blocks — prompt templates, chains, memory, tools, and agents — that simplify the development of complex AI systems. Rather than directly calling an LLM API, LangChain enables you to structure and combine these components into production workflows.

Core Concepts

Chains

Sequences of LLM calls with intermediate outputs. Enable multi-step reasoning and complex workflows by piping results from one call into the next.

Memory

Stateful conversation history. Maintains context across multiple turns, enabling the LLM to understand the full conversation history and respond coherently.

Tools & Agents

Enable LLMs to take actions. Agents reason about which tools to use and in what order, implementing the ReAct (Reasoning + Acting) loop for dynamic problem-solving.

Output Parsing

Structures unstructured LLM output. Enforces format constraints and extracts data into JSON, dataclasses, or other typed formats for reliable downstream processing.

LangGraph: Stateful Workflows

LangGraph extends LangChain with explicit state management and graph-based control flow. It enables you to define complex workflows as directed graphs where nodes represent computational steps and edges represent transitions. Key benefits:

  • Explicit State: All state transitions are visible and debuggable. State flows through the graph, and each node receives the current state as input.
  • Conditional Edges: Routes decisions dynamically based on the current state, enabling branching logic like "if validation fails, reroute to correction step".
  • Cycles & Loops: Support iterative refinement. A node can loop back to itself or route to an earlier step, enabling multi-turn reasoning and progressive improvement.
  • Human-in-the-Loop: Pause execution and wait for human input at any step, then resume with the approved state. Critical for high-stakes applications.

Interview Prep: Key Questions

Q: When would you use LangChain vs. calling the Anthropic API directly?

A: Use LangChain when building complex multi-step workflows. LangChain handles composition, memory management, and tool orchestration. Use the API directly for simple single-turn interactions where these abstractions add overhead.

Q: What is the ReAct pattern and why is it important?

A: ReAct (Reasoning + Acting) is an agent loop where the LLM reasons about the next step, takes an action (calls a tool), observes the result, and repeats. It breaks complex problems into smaller steps, making reasoning transparent and enabling dynamic adaptation.

Q: How does LangGraph differ from LangChain chains?

A: Chains execute a fixed sequence of steps. LangGraph enables dynamic routing based on state, cycles for iteration, and explicit human pauses. Use chains for linear workflows; use LangGraph when you need branching, loops, or complex state management.
