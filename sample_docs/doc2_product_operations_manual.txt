1. PRODUCT OVERVIEW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Clarivex is a cloud-based data analytics platform serving mid-market and enterprise clients across the financial services, healthcare, and retail sectors. The platform ingests structured data from client databases, applies configurable transformation pipelines, and surfaces insights via an embedded BI dashboard and a REST API. Clarivex is deployed on AWS in three regions: us-east-1 (primary), eu-west-1 (GDPR zone), and ap-southeast-1 (APAC zone).

The platform is organised around three core modules: DataBridge (ingestion and transformation), InsightEngine (analytics and ML scoring), and ReportFlow (scheduled reporting and alerting). Clients interact with the platform through a React-based web portal (portal.clarivex.io) or via the Clarivex REST API v3 (documented at api.clarivex.io/v3/docs).

3. DATABRIDGE MODULE — CONFIGURATION AND LIMITS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

DataBridge supports ingestion from the following source types: PostgreSQL, MySQL, Microsoft SQL Server, Snowflake, Google BigQuery, Amazon S3 (CSV and Parquet), and Salesforce (via OAuth 2.0). MongoDB and Oracle DB connectors are on the Athena-v3 roadmap and are not currently supported.

3.1 Ingestion Frequency

Starter tier: Batch ingestion only, minimum interval 24 hours.
Professional tier: Batch ingestion minimum interval 1 hour; near-real-time streaming via webhook push (latency ~5 minutes) available as an add-on (SKU: CLX-PRO-STREAM, USD 399/month).
Enterprise tier: Real-time streaming (latency <30 seconds) included by default via Kinesis Firehose integration.

3.2 Transformation Pipeline Limits

Each pipeline step applies one transformation rule to one or more columns. Supported rule types include: type casting, regex extraction, null imputation (mean, median, mode, or constant), deduplication by primary key, join (inner, left, right), aggregate (sum, count, avg, min, max), and pivot.

Starter: Maximum 10 rules per pipeline, 3 pipelines total.
Professional: Maximum 50 rules per pipeline, 20 pipelines total.
Enterprise: No hard limit; performance SLA applies (pipeline execution must complete within the agreed processing window specified in the enterprise contract).

3.3 Error Handling in Pipelines

When a transformation rule fails (e.g., type cast fails due to unexpected null values), DataBridge applies the configured error policy: SKIP (discard the failing row and continue), HALT (stop pipeline execution and raise alert ERR-DB-001), or QUARANTINE (route the failing row to a designated quarantine table for manual review). The default policy is QUARANTINE. Error policy is configurable per pipeline, not per rule.

4. INSIGHTENGINE MODULE — ML SCORING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

InsightEngine provides three pre-built ML scoring models: ChurnPredictor, RevenueForecaster, and AnomalyDetector. Each model is retrained on the first Sunday of each month using the client's accumulated data. A minimum of 500 rows of historical data is required before a model will produce predictions; below this threshold, the API returns error code INS-001 ("Insufficient training data").

ChurnPredictor outputs a score between 0.0 and 1.0 representing the 30-day churn probability for each customer record. Scores above 0.75 are flagged as high risk and automatically trigger a ReportFlow alert to the configured recipient list. The model uses a gradient-boosted tree (XGBoost) with 14 input features; clients must map at least 8 of the 14 features for the model to activate.

RevenueForecaster produces a 90-day revenue projection per client segment. It uses a seasonal ARIMA model with automatic parameter selection (auto-ARIMA). Forecast accuracy (MAPE) on the internal validation set averages 8.3%. Forecasts are recalculated every 24 hours regardless of ingestion frequency.

AnomalyDetector uses an Isolation Forest algorithm to flag statistical outliers in any numeric column. Sensitivity is configurable from 1 (low) to 5 (high). At sensitivity 5, expect approximately 2–3% of rows to be flagged as anomalies in a healthy dataset; at sensitivity 1, expect less than 0.1%.

5. TROUBLESHOOTING GUIDE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

5.1 Error Code Reference

ERR-DB-001: Pipeline execution halted. Check the pipeline's error policy configuration. If policy is HALT, review the quarantine table for the failing row. Switch to QUARANTINE policy to allow continuation. Contact support if the quarantine table is empty (indicates a DataBridge internal error).

ERR-DB-002: Source connector authentication failure. Re-validate credentials in the DataBridge connector settings. For Salesforce connectors, OAuth tokens expire after 90 days; re-authorise via the portal under Settings > Connectors > Salesforce > Re-authorise.

ERR-DB-003: Storage quota exceeded. The client's allocated storage is full. Starter and Professional tier clients must delete old datasets or upgrade. Enterprise clients should contact their CSM to request a storage allocation increase (typically provisioned within 1 business day).

INS-001: Insufficient training data. The InsightEngine model requires at least 500 rows. Ensure DataBridge has successfully ingested the required historical data. Check the ingestion log (portal: DataBridge > Pipelines > [Pipeline Name] > Logs) for failed batches.

INS-002: Feature mapping incomplete. Fewer than 8 of the 14 required ChurnPredictor features are mapped. Navigate to InsightEngine > ChurnPredictor > Feature Mapping to complete the configuration.

REP-001: Report delivery failure. Scheduled report failed to deliver to the configured email or webhook. Check recipient email is not on the platform suppression list. For webhook delivery, verify the endpoint returns HTTP 200 within 10 seconds; timeouts cause REP-001.

5.2 Common User Issues

Issue: User cannot log in after password reset.
Resolution: Password reset links expire after 2 hours. If the link has expired, the user must request a new reset. Administrators can force a new reset from the Admin Panel > Users > [User] > Force Password Reset. Do not share admin credentials with end users.

Issue: Dashboard loading slowly or timing out.
Resolution: Large dashboards with more than 15 widgets and date ranges exceeding 12 months are known to cause timeouts on the Starter tier. Recommended action: reduce the date range to 3 months, or split the dashboard into multiple smaller ones. Professional and Enterprise clients can enable dashboard query caching (Settings > Performance > Enable Query Cache) to reduce load times by up to 60%.

Issue: API rate limit exceeded (HTTP 429).
Resolution: Each tier has a monthly API call limit, not a per-second rate limit. HTTP 429 indicates the monthly quota is exhausted. The quota resets on the 1st of each calendar month. Clients can purchase an API overage pack (SKU: CLX-API-OVR, USD 15 per additional 10,000 calls) or upgrade their tier.

6. SERVICE LEVEL AGREEMENT (SLA) AND PENALTIES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Uptime is measured monthly as: (Total minutes in month − Downtime minutes) / Total minutes in month × 100%.

Scheduled maintenance windows (announced 72 hours in advance via email and status.clarivex.io) are excluded from downtime calculations. Emergency maintenance windows are not excluded.

SLA breach thresholds and service credits:

Starter (SLA: 99.5%):
Uptime 99.0%–99.4%: 5% credit on monthly fee.
Uptime 95.0%–98.9%: 15% credit on monthly fee.
Uptime below 95.0%: 30% credit on monthly fee.

Professional (SLA: 99.8%):
Uptime 99.5%–99.7%: 10% credit on monthly fee.
Uptime 99.0%–99.4%: 25% credit on monthly fee.
Uptime below 99.0%: 50% credit on monthly fee.

Enterprise (SLA: 99.95%):
Uptime 99.8%–99.94%: 10% credit on monthly fee.
Uptime 99.5%–99.79%: 25% credit on monthly fee.
Uptime below 99.5%: 50% credit on monthly fee, plus the right to terminate with 30 days notice without early termination fees.

Credits must be claimed within 30 days of the SLA breach by submitting a credit request via the support portal (support.clarivex.io). Credits are applied to the following month's invoice and are not redeemable as cash. Maximum credits per month are capped at 50% of the monthly fee regardless of tier or severity.

7. INCIDENT ESCALATION PROCEDURE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Incidents are classified into four priority levels based on business impact.

P1 — Critical: Platform completely unavailable or data loss occurring. All tiers: submit via support portal and call the emergency hotline (+65 6123 4567). Enterprise clients: dedicated P1 response within 15 minutes. Professional clients: response within 2 hours. Starter clients: response next business day.

P2 — High: Major feature unavailable (e.g., DataBridge ingestion stopped, InsightEngine scoring offline) but platform partially operational. Response time: Enterprise 1 hour, Professional 4 hours, Starter next business day.

P3 — Medium: Non-critical feature degraded (e.g., slow dashboard load, delayed reports). Response time: Enterprise 4 hours, Professional next business day, Starter within 3 business days.

P4 — Low: Cosmetic issues, documentation requests, feature inquiries. Response time: all tiers within 5 business days.

Escalation path: Front-line Support Engineer → Senior Support Engineer (if unresolved after 2 × SLA response time) → Platform Operations Lead → VP of Engineering (P1 only, if unresolved after 4 hours).

All P1 and P2 incidents are logged in the internal incident tracker (Jira project: CLX-OPS). Post-incident reviews (PIR) are mandatory for all P1 incidents and must be completed within 5 business days of resolution. The PIR template is available on the internal wiki at wiki.clarivex.internal/PIR-template.
