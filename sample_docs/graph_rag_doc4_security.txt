NEXUS CORP — SECURITY VULNERABILITY BULLETIN
=============================================

CLASSIFICATION: URGENT — Requires immediate action from all affected teams.
Issued by: Marcus Rodriguez, Chief Security Officer
Date: Q1 2025

This bulletin documents critical and high-severity vulnerabilities identified in technologies currently in use at Nexus Corp. Teams using the affected technology versions must apply patches within the timelines stated below.

---

CVE-2024-7891 — CRITICAL
--------------------------
Technology:   Kubernetes
Versions:     v1.26, v1.27, v1.28 (all patch levels)
Severity:     CRITICAL (CVSS 9.8)
Type:         Privilege escalation via container runtime escape
Description:  A flaw in the Kubernetes API server allows an unauthenticated attacker within the cluster network to escalate privileges to cluster-admin level. Exploitation does not require existing access. Proof-of-concept exploit code is publicly available.
Action:       Upgrade to Kubernetes v1.29.2 or later immediately.
Deadline:     72 hours from date of this bulletin.
Contact:      Security Team (Marcus Rodriguez or Nina Okafor) for upgrade assistance.

---

CVE-2024-3412 — HIGH
----------------------
Technology:   PostgreSQL
Versions:     v14.x, v15.0 – v15.3
Severity:     HIGH (CVSS 8.1)
Type:         SQL injection via crafted COPY FROM command
Description:  Insufficient input sanitisation in the COPY FROM PROGRAM command allows authenticated users to inject arbitrary SQL. This can lead to full database compromise including data exfiltration or deletion.
Action:       Upgrade to PostgreSQL v15.4 or later. Note: v15.4 is already deployed in some services — verify all instances are on v15.4 specifically.
Deadline:     7 days from date of this bulletin.
Contact:      Security Team for verification scripts.

---

CVE-2024-9103 — MEDIUM
------------------------
Technology:   MLflow
Versions:     v2.6.x, v2.7.x
Severity:     MEDIUM (CVSS 6.5)
Type:         Arbitrary file write via crafted model artefact
Description:  MLflow's model loading mechanism does not validate file paths in model artefact packages. A malicious artefact can overwrite arbitrary files on the tracking server.
Action:       Upgrade to MLflow v2.8 or later. If upgrade is not immediately possible, restrict artefact upload permissions to trusted users only.
Deadline:     30 days from date of this bulletin.
Contact:      Security Team for interim mitigation steps.
